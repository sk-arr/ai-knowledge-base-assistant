# AI企业知识库问答助手

一个面向企业内部资料查询、制度问答和文档检索场景的轻量化 AI 应用。

项目把「上传资料 → 文本切分 → **混合检索** → **大模型生成** → 来源展示」的知识库问答流程工具化，完整落地了 RAG（检索增强生成）的核心链路，并做了相关度门槛与优雅降级，兼顾效果与可演示性。

> **稳定演示优先**：配置 `ANTHROPIC_API_KEY` 即用真实大模型生成；未配置或调用失败时自动回退本地 Mock 回答，保证任何环境下都能跑、能演示。

## 功能列表

- 支持**同时上传多个** TXT / MD 文档，跨文档统一检索，引用标明来源
- 自动解析并按结构切分为知识片段
- **混合检索**：BM25（字面）+ bge 向量（语义）加权召回 Top-K 相关片段
- **真实大模型生成**：把检索片段作为上下文传给 Claude，生成「简要结论 / 依据摘要 / 建议下一步」；无 API Key 时回退 Mock
- **相关度门槛防幻觉**：检索不到足够相关的内容时，直接回答「文档中未找到相关内容」，不编造依据
- 展示引用来源片段与相关分数，便于人工复核
- 支持最近 10 条问答历史
- 支持导出当前问答结果为 Markdown 文件

## 检索与生成流程

```text
问题
 ├─ BM25 打分（jieba 分词 + 停用词过滤 + IDF 加权）───┐
 └─ 向量打分（bge-small-zh 余弦相似度，理解语义）──────┤
                                                    ├─ 归一化加权融合（字面 0.4 / 语义 0.6）→ Top-K
                                                    │
 相关度门槛判断（语义≥0.40 或 BM25 明显命中）
   ├─ 不相关 → 回「未找到相关内容」（防幻觉，省调用）
   └─ 相关   → 片段作为上下文 → Claude 生成结构化回答
                                  └─（无 Key / 失败）→ 回退 Mock
```

- **为什么混合检索**：BM25 保证关键词精确命中，向量检索能召回「换了说法」的内容（如「报销」≈「费用申请」），二者互补是工业界主流做法。
- **为什么有门槛**：真正的 RAG 要能判断「文档里根本没有答案」，而不是硬把最相似的片段当证据——这是防幻觉的关键。

## 技术栈

- Python / Streamlit
- 检索：jieba 分词、rank_bm25、sentence-transformers（`BAAI/bge-small-zh-v1.5`）、numpy
- 生成：anthropic 官方 SDK（`claude-opus-4-8`，结构化输出）
- 存储与导出：JSON 本地历史、Markdown 导出

## 运行方式

安装依赖：

```bash
pip install -r requirements.txt
```

> 首次运行会自动下载 bge 向量模型（约 100MB）。`sentence-transformers` 依赖 PyTorch，包较大；因已做降级处理，资源受限时也可只跑 BM25 + Mock。

启动项目：

```bash
streamlit run app.py
```

### 环境变量（可选）

| 变量 | 作用 |
|---|---|
| `ANTHROPIC_API_KEY` | 设置后启用真实 Claude 生成；不设则用 Mock 演示 |
| `KB_MODEL` | 覆盖默认模型（默认 `claude-opus-4-8`） |
| `KB_FORCE_MOCK=1` | 强制使用 Mock，便于离线演示 |

## 如何测试

1. 页面上传 `sample_docs/company_ai_policy.md`
2. 输入相关问题，例如：`使用 AI 工具时哪些敏感信息不能上传？`
3. 点击「检索资料并生成回答」，查看回答、依据摘要和引用片段
4. 试试无关问题（如「公司食堂几点开饭？」），验证「未找到相关内容」的防幻觉行为
5. 点击 Markdown 导出按钮

## 项目亮点

- 完整落地 RAG 链路：切分 → 混合检索 → 大模型生成 → 引用展示
- **混合检索**兼顾字面与语义，检索质量优于单一关键词匹配
- **相关度门槛**从源头防幻觉，无依据不作答
- **优雅降级**：真实大模型 ↔ 本地 Mock 自动切换，演示稳定、便于在线部署
- UI 采用企业知识库控制台风格，代码分层清晰
